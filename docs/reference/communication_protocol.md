# 鲁班猫到 F407 通信协议说明

## 1. 通信目标

鲁班猫负责生成三个苗盘的视觉识别矩阵，并将矩阵发送给 F407。F407 接收后负责坐标转换、五连杆逆解、矩阵滚动更新和执行机构控制。

通信内容当前只包含鲁班猫输出的前 8 列数据：

```text
tray_id, col, row, class_id, confidence, u, v, z
```

## 2. 通信方式

推荐通信链路：

```text
鲁班猫 5 网口 <-> 以太网 TCP/IP <-> F407 + W5500
```

TCP 的优点是可靠、有连接状态，适合先做调试。后续如果 F407 侧需要更低开销，可以再评估 UDP 或自定义串口协议。

## 3. 数据发送频率

建议采用“苗盘到位后发送一次完整矩阵”的方式，而不是持续高频发送。

典型流程：

1. F407 判断 1、2、3 号苗盘到位。
2. F407 请求或等待鲁班猫识别。
3. 鲁班猫完成识别。
4. 鲁班猫发送 150 行完整矩阵。
5. F407 校验并更新内部矩阵。
6. F407 执行后续分拣控制。

## 4. 矩阵字段

每个穴位一条记录，字段如下：

| 字段 | 类型建议 | 说明 |
| --- | --- | --- |
| `tray_id` | `uint8` | 苗盘序号，`1..3` |
| `col` | `uint8` | 穴位列坐标，`1..5` |
| `row` | `uint8` | 穴位行坐标，`1..10` |
| `class_id` | `uint8` | 类别，`0/1/2` |
| `confidence` | `float32` | 类别置信度，`0.0..1.0` |
| `u` | `float32` | 像素横坐标 |
| `v` | `float32` | 像素纵坐标 |
| `z` | `float32` | 深度值，单位 mm |

每次完整发送：

```text
3 个苗盘 x 50 个穴位 = 150 条记录
```

ROS 2 内部对应消息：

- `sorting_interfaces/msg/TrayCell`：单个穴位信息。
- `sorting_interfaces/msg/TrayMatrix`：一次完整视觉识别结果，正常包含 150 个 `TrayCell`。

## 5. 推荐文本协议

早期调试阶段建议先用文本协议，方便串口助手、网络调试助手和日志查看。

一帧完整矩阵示例：

```text
START frame_id=12 count=150
1,1,1,1,0.98,325.4,218.7,612.0
1,2,1,2,0.91,371.2,219.3,610.8
...
3,5,10,0,0.87,918.5,640.1,628.2
END checksum=12345
```

当前 ROS 2 调试节点 `matrix_protocol_printer` 会订阅 `/sorting/tray_matrix`，并按上述文本格式打印完整帧。该节点暂时不连接 F407，只用于确认协议内容。

`matrix_tcp_sender` 会使用同样的文本格式，通过 TCP 发送完整帧。默认连接 `127.0.0.1:9000`，现场可通过 ROS 参数改为 F407/W5500 的 IP 和端口。

字段说明：

- `START`：一帧开始。
- `frame_id`：鲁班猫侧递增编号，用于排查丢帧。
- `count`：本帧记录数，正常为 150。
- 中间每行是一条穴位记录。
- `END`：一帧结束。
- `checksum`：简单累加校验值。只累加 150 行 CSV 数据行的原始 UTF-8 字节，对 65536 取模。

checksum 范围必须严格一致：

```text
checksum = sum(bytes(line_1) + bytes(line_2) + ... + bytes(line_150)) % 65536
```

不参与 checksum 的内容：

- `START` 行。
- `END` 行。
- 每行之间的换行符。
- 文件或 TCP 流末尾额外换行符。

## 6. 推荐二进制协议

系统稳定后可以改为二进制协议，减少带宽和解析开销。

建议帧结构：

| 字段 | 大小 | 说明 |
| --- | --- | --- |
| `magic` | 2 bytes | 固定帧头，例如 `0xAA55` |
| `version` | 1 byte | 协议版本 |
| `frame_id` | 2 bytes | 帧编号 |
| `count` | 2 bytes | 记录数量，正常为 150 |
| `payload` | N bytes | 150 条穴位记录 |
| `crc16` | 2 bytes | CRC 校验 |

单条穴位记录建议：

| 字段 | 类型 |
| --- | --- |
| `tray_id` | `uint8` |
| `col` | `uint8` |
| `row` | `uint8` |
| `class_id` | `uint8` |
| `confidence` | `float32` |
| `u` | `float32` |
| `v` | `float32` |
| `z` | `float32` |

## 7. 错误处理

F407 收到数据后建议检查：

- `count` 是否为 150。
- `tray_id` 是否只包含 `1..3`。
- `col` 是否在 `1..5`。
- `row` 是否在 `1..10`。
- `class_id` 是否在 `0..2`。
- `confidence` 是否在 `0.0..1.0`。
- `z` 是否在合理深度范围内。
- 校验值是否正确。

如果校验失败，F407 不应更新内部矩阵，应要求鲁班猫重发或等待下一帧。

## 8. 后续待定

- 具体使用 TCP 客户端还是 TCP 服务端，需要与 F407 程序确认。
- 是否需要 F407 主动请求识别，需要结合现场控制流程确定。
- 是否需要 F407 回传执行状态，需要后续定义状态帧。
