# 视觉算法说明

## 1. 输入与输出

输入：

- D435iF RGB 图像。
- D435iF 深度图。
- 相机内参。

输出：

```text
tray_id, col, row, class_id, confidence, u, v, z
```

三个苗盘共输出 150 条记录。

## 2. 识别流程

整体流程：

1. 获取 RGB-D 图像。
2. 识别三个苗盘外框。
3. 对每个苗盘做透视矫正。
4. 在矫正后的苗盘图像中生成 `5 x 10` 网格。
5. 计算每个穴位中心点。
6. 根据中心点裁剪单穴 ROI。
7. 使用深度和图像特征判断类别。
8. 生成矩阵并发送给 F407。

当前视觉开发主线优先走离线样本调试：

```text
收集样本图片
-> 离线读取图片
-> 检测苗盘外框
-> 透视矫正
-> 生成 5x10 网格
-> 输出调试图片
-> 再迁移到实时 D435iF/RGB-D
```

`offline_tray_debug` 是这条主线的第一版工具。它读取样本图片，自动检测矩形
候选或读取配置四角点，输出外框、透视矫正和 `5 x 10` 网格调试图。
当前自动检测优先使用“深色苗盘区域横向投影”方案：先分割深色塑料苗盘，
再按横向暗像素分布切分左、中、右三个候选，最后分别计算外接矩形和网格。
如果该方案在新样本中不稳定，仍可临时使用配置四角点验证透视矫正流程。
同时它会输出 `*_cells.csv`，字段顺序为
`tray_id,col,row,class_id,confidence,u,v,z`。当前离线照片没有深度图，
分类算法也还没接入，所以 `class_id/confidence/z` 先是占位值；这一文件
主要用于检查 150 个穴位编号和 `u/v` 图像坐标是否稳定。

当前代码中的第一版真实相机节点是 `real_matrix_publisher`。它先跳过自动
苗盘外框识别，使用配置文件里的 `tray_1_roi`、`tray_2_roi`、`tray_3_roi`
作为三个苗盘矩形区域，再按 `5 x 10` 规则网格生成 150 个穴位中心点。
这样可以先验证相机、网格编号、深度采样和通信链路。它是早期联调工具，
不作为最终长期依赖。

为了便于现场调整 ROI，当前还加入了 `grid_debug_publisher`。它订阅 RGB
图像，在图像上画出三个 ROI、`5 x 10` 网格线和穴位中心点，并发布到
`/sorting/debug/grid_image`。现场应先用这个调试图像确认中心点落在穴位
附近，再继续验证分类和深度。

## 3. 苗盘外框识别

苗盘外框可以先采用传统图像方法：

- 图像预处理。
- 边缘检测或颜色/亮度分割。
- 轮廓查找。
- 筛选近似矩形区域。
- 按面积、长宽比和位置过滤候选框。

识别到外框后，获得四个角点，并进行透视变换。

如果传统方法在真实环境下不稳定，再考虑使用目标检测模型识别苗盘区域。

## 4. 透视矫正

透视矫正目标是把苗盘拉成标准正视图，使穴位网格可以按固定行列生成。

每个苗盘单独矫正，输出固定尺寸图像。固定尺寸需要根据真实苗盘比例确定。

建议保证：

- 矫正后左上角对应 `row=1, col=1`。
- 矫正后列方向为图像横向。
- 矫正后行方向为图像纵向。

## 5. 网格生成

苗盘固定为：

```text
col = 1..5
row = 1..10
```

每个穴位中心点由矫正图中的规则网格计算得到，再通过透视变换的逆映射换回原图像素坐标，得到 `u`、`v`。

深度值 `z` 从原始深度图中读取。建议不要只取单个像素，可以取中心点附近小窗口的中位数，减少深度噪声。

## 6. ROI 裁剪

每个穴位以中心点为中心生成 ROI。文档中暂定 ROI 直径为 42 mm。

实现时需要把 42 mm 转换到矫正图像中的像素尺寸，或者直接按矫正后网格单元的比例裁剪。

ROI 应避免覆盖相邻穴位，必要时使用圆形或内缩矩形区域。

## 7. 类别判定

当前类别：

```text
0 = 空穴或无有效苗
1 = 好苗
2 = 弱苗或未出土苗
```

建议先按以下逻辑实现：

1. 使用深度判断是否为空穴或异常穴位。
2. 如果不是空穴，计算叶片投影面积占 ROI 有效区域的比例。
3. 如果叶片面积占比大于阈值，则判为 1 类。
4. 如果叶片面积占比小于等于阈值，则判为 2 类。

初始阈值：

```text
leaf_area_ratio_threshold = 0.20
```

这个阈值需要通过真实样本调试。

`real_matrix_publisher` 当前使用绿色面积占比作为临时规则：

- `green_ratio >= leaf_area_ratio_threshold`：暂判为 1 类。
- `weak_area_ratio_threshold <= green_ratio < leaf_area_ratio_threshold`：暂判为 2 类。
- `green_ratio < weak_area_ratio_threshold`：暂判为 0 类。

该规则主要服务早期联调和小球/绿叶显著样本测试，不是最终分类算法。

## 8. 前期小球替代实验

前期可以使用不同颜色小球代替不同类别苗。

小球实验阶段可以先使用颜色分割：

- 颜色 A 表示 1 类。
- 颜色 B 表示 2 类。
- 无小球或空位表示 0 类。

小球实验的目标不是最终识别算法，而是验证：

- 苗盘定位是否稳定。
- 网格编号是否正确。
- 矩阵输出是否正确。
- 鲁班猫到 F407 通信是否稳定。
- F407 运动控制流程是否正确。

## 9. 置信度

`confidence` 建议表示当前分类结果可信程度，范围为 `0.0..1.0`。

早期可以使用规则置信度：

- 图像和深度都稳定时，给较高置信度。
- 深度缺失、ROI 边缘、分割面积接近阈值时，降低置信度。

后续如果使用模型分类，则使用模型输出置信度。

## 10. 待验证问题

- 苗盘边框颜色、反光和遮挡对外框识别的影响。
- D435iF 深度在培养土、叶片、小球上的稳定性。
- 42 mm ROI 是否适合实际穴位尺寸。
- 20% 叶片面积阈值是否适合不同生长阶段。
- 真实苗和小球实验之间的算法差异。
