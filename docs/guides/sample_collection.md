# 样本采集说明

本文档用于指导“离线视觉算法调试”的样本图片采集。当前视觉主线是：

```text
收集样本图片
-> 离线读取图片
-> 检测苗盘外框
-> 透视矫正
-> 生成 5x10 网格
-> 输出调试图片
-> 再迁移到实时 D435iF/RGB-D
```

固定 ROI 只作为早期调试辅助，不作为最终长期依赖。

## 1. 样本目录建议

建议在项目根目录下建立本地样本目录：

```text
samples/
  raw/
  debug/
```

说明：

- `samples/raw/`：保存原始拍摄图片。
- `samples/debug_/`：保存离线脚本输出的调试图片。
- 样本图片不建议直接提交到 Git，除非后续选少量小图作为固定测试样例。

## 2. 图片命名建议

建议使用能看懂的文件名：

```text
samples/raw/20260428_usb_front_001.jpg
samples/raw/20260428_usb_left_offset_001.jpg
samples/raw/20260428_d435if_top_001.jpg
```

命名包含：

- 日期。
- 相机类型，例如 `usb`、`d435if`。
- 拍摄角度或场景，例如 `front`、`top`、`left_offset`。
- 编号。

## 3. 推荐采集内容

每组样本建议至少包含：

- 苗盘居中、无遮挡、光照均匀的图片。
- 苗盘向左偏一点的图片。
- 苗盘向右偏一点的图片。
- 苗盘轻微旋转的图片。
- 背景有干扰物的图片。
- 光照偏亮和偏暗的图片。

如果现场有三条传送带，也建议分别拍：

- 单个苗盘。
- 两个苗盘。
- 三个苗盘同时在画面内。

## 4. 拍摄注意事项

- 尽量保证苗盘完整进入画面。
- 苗盘四个角尽量不要被手、支架、传送带边缘遮挡。
- 尽量使用稳定补光，减少强反光和阴影。
- 如果用手机拍样本，尽量保持镜头平稳，不要过度倾斜。
- 如果用 USB 相机或 D435iF 截图，尽量记录分辨率和相机安装位置。

## 5. 标注四角点的临时方案

自动检测一开始可能不稳定，因此第一版离线脚本允许从配置文件读取四角点。

四角点顺序统一为：

```text
top_left
top_right
bottom_right
bottom_left
```

示例：

```yaml
trays:
  - tray_id: 1
    corners:
      - [120, 80]
      - [430, 76]
      - [445, 690]
      - [105, 700]
```

这个方案只是过渡入口，目的是先验证透视矫正和 `5 x 10` 网格生成。
后续主线仍然是自动检测苗盘外框。

## 6. 离线调试输出

离线脚本应输出：

- 原图上的苗盘外框和网格反投影图。
- 每个苗盘的透视矫正图。
- 每个苗盘的透视矫正后 `5 x 10` 网格图。
- 自动检测失败时的候选外框图。

这些调试图片用于判断：

- 外框是否找准。
- 四角点顺序是否正确。
- 透视矫正后苗盘是否横平竖直。
- 5 列 10 行网格是否落在穴位中心附近。
